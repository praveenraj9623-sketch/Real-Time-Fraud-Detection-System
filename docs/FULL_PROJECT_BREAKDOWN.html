Problem Statement

Credit-card fraud detection is a rare-event classification problem. The goal is not only to produce a model, but to show how a data science model would be wrapped in a usable monitoring system. This project demonstrates:

  • Feature engineering for batch and real-time inference.
  • Imbalanced classification using SMOTE and XGBoost.
  • Anomaly detection with Isolation Forest as a comparison model.
  • Threshold tuning with precision, recall, F1, AUC, AUPRC, and business-cost logic.
  • FastAPI scoring endpoints with SHAP explanation output.
  • Kafka and Spark Structured Streaming simulation.
  • MongoDB alert storage and Streamlit monitoring dashboard.

Repository Map

PathPurposeWhat reviewers should inspect
app.py Streamlit dashboard with live feed, analytics, model performance, and risk investigation tabs. UI logic, active-threshold model metrics, event-time analytics, MongoDB interactions.
src/api/main.py FastAPI scoring service. Feature engineering consistency, probability clipping, SHAP output, decision thresholds.
src/processing/feature_engineering.py Creates processed features from the Kaggle CSV. Feature parity between batch and inference paths.
src/modeling/train_batch.py Trains XGBoost and Isolation Forest with MLflow tracking. SMOTE handling, metrics, artifact writing, validation split, model registration attempt.
src/modeling/threshold_tuner.py Finds tuned thresholds and writes threshold artifacts. Precision-recall tradeoff, business-cost threshold, active threshold summary.
src/storage/mongodb_client.py MongoDB insert/query/analytics wrapper. Alert normalization, merchant risk summary, hourly fraud analytics, event-time handling.
src/streaming/ Kafka producer, Spark consumer, and local demo generator. Streaming data flow, score URL handling, MongoDB alert insert behavior.
src/utils/risk.py Shared risk and decision helper functions. Decision bands, probability/risk clipping, timestamp normalization.
src/utils/demo_selection.py Lightweight demo alert sampler. Ensures demo alerts use real model scores while balancing decision and amount variety.
tests/ Fast regression tests. Risk logic, active-threshold confusion consistency, event-time timeline, merchant block rate.

Modeling Breakdown

ModelRoleStrengthDashboard handling
XGBoost Classifier Main supervised fraud classifier. Strong performance on rare fraud classification when threshold is tuned. Displayed as champion model in the fraud monitor's Model Performance tab.
Isolation Forest Unsupervised anomaly detector comparison. Useful baseline when labels are weak or unavailable. Shown in model comparison with AUC/AUPRC and threshold-related metrics where available.

AUPRC is highlighted because fraud is highly imbalanced. Accuracy can look excellent even when a model misses most fraud cases.

Recent Logic Fixes Included In This Version

  • Model Performance table uses active-threshold XGBoost metrics rather than stale 0.90 metrics.
  • Dynamic confusion matrix matches active-threshold table counts.
  • Threshold chart marks best F1 and best business-cost thresholds separately.
  • Alert analytics use event_time_utc, then transaction_time_utc, then stored_at_utc.
  • Night-time alerts use simulated transaction event time and define night as 10 PM to 6 AM.
  • Merchant intelligence includes block rate, review count, median risk score, and improved sorting.

Limitations

  • The raw dataset must be downloaded separately from Kaggle.
  • The review zip cannot retrain models without raw data and binary dependencies.
  • V1-V28 are anonymized PCA features, so explanations are mathematical rather than direct business narratives.
  • Real-time streaming requires Kafka, Spark, and Java, which can be heavier on Windows.
  • Dashboard demo alerts are generated from real scored dataset rows, but the event timestamps and merchant names are simulated for portfolio demonstration.
