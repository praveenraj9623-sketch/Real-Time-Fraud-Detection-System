Real-Time Fraud Detection System

A portfolio project that simulates how a bank or payment company could score credit-card transactions, block high-risk activity, store fraud alerts, and give analysts a dashboard for monitoring and investigation.

Python XGBoost Isolation Forest SMOTE FastAPI MongoDB Streamlit Kafka/Spark architecture SHAP

1. What This Project Is For

Credit card fraud is rare, but expensive. A simple accuracy metric is misleading because a model can be highly accurate by predicting almost every transaction as legitimate. This project focuses on the metrics that matter for rare fraud: AUPRC, recall, precision, F1, false positives, and false negatives.

The dashboard is designed for a recruiter or interviewer to see the full machine-learning lifecycle: feature engineering, class imbalance handling, model training, threshold tuning, API scoring, alert storage, and analyst investigation.

4. Risk Decision Logic

The project now uses clear bands. Probabilities are always clipped to 0-1 and risk scores to 0-100.

ConditionDecisionMeaning
probability < save_thresholdAPPROVEDNot saved as an alert unless explicitly requested.
save_threshold <= probability < 0.50REVIEWMedium-low risk. Put in a monitoring or review queue.
0.50 <= probability < model_thresholdFLAGGEDSuspicious. Hold for step-up authentication or analyst review.
probability >= model_thresholdBLOCKEDHigh-risk. Block and investigate.

5. What app.py Displays

Live Feed

Shows recent saved alerts from MongoDB, KPI counts, average risk gauge, blocked amount, highest-risk transactions, and a formatted alert table.

Analytics

Shows alert count by hour, amount distribution, merchant risk scatter plot, and high-risk merchant ranking with average/max risk instead of summed probability.

Model Performance

Shows XGBoost vs Isolation Forest metrics, confusion counts, false positives, missed fraud, threshold tradeoff, PR curve, ROC curve, and AUPRC explanation.

Risk Investigation

Lets the analyst choose or paste a transaction_id, then view decision, risk, merchant, amount, timestamp, SHAP drivers, and the full MongoDB alert JSON.

7. Beginner Command Prompt Commands

Open Command Prompt in the project folder. Activate the virtual environment first:

cd /d "C:\Users\admin\Desktop\Real-Time Fraud Detection System"
venv\Scripts\activate.bat

Install/repair packages:

python -m pip install --upgrade pip setuptools wheel
pip install -r requirements.txt

Build features, train, and tune:

python -m src.processing.feature_engineering
python -m src.modeling.train_batch
python -m src.modeling.threshold_tuner

Start FastAPI in one Command Prompt:

cd /d "C:\Users\admin\Desktop\Real-Time Fraud Detection System"
venv\Scripts\activate.bat
uvicorn src.api.main:app --host 127.0.0.1 --port 8001 --reload

Start Streamlit in another Command Prompt:

cd /d "C:\Users\admin\Desktop\Real-Time Fraud Detection System"
venv\Scripts\activate.bat
streamlit run app.py --server.port 8502
Open the API at http://127.0.0.1:8001/docs and the dashboard at http://127.0.0.1:8502. If the dashboard is empty, click Generate Demo Alerts.

8. Assumptions and Limitations

  • The Kaggle dataset has anonymized PCA features, so explanations are mathematical rather than direct business explanations.
  • The local demo generator simulates streaming alerts for easy Windows demos. Kafka and Spark files are still present for the full architecture.
  • The model is trained on a public dataset, not real bank production data.
  • Threshold tuning includes F1 and business-cost views, but business costs are configurable assumptions.
  • The zip package excludes the virtual environment and raw Kaggle CSV to keep the package portable.
