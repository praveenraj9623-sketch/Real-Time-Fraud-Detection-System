Real-Time Fraud Detection System

Full project breakdown for portfolio review and LLM cross-checking. This project simulates a payment-company fraud workflow: feature engineering, imbalanced model training, threshold tuning, API scoring, alert storage, real-time-style demo generation, and a Streamlit analyst dashboard.

The main production-style flow is Kafka producer to Spark streaming consumer to FastAPI scoring to MongoDB alerts to Streamlit monitoring. Because Windows/Docker/Kafka setup can be heavy for demos, the dashboard also includes a local demo generator that scores real rows from creditcard.csv and inserts a realistic mixed set of review, flagged, and blocked alerts into MongoDB.

6. Active Risk Decision Logic

ConditionDecisionDashboard Meaning
probability < save_thresholdAPPROVEDNot normally saved as a dashboard alert.
save_threshold <= probability < 0.50REVIEWMedium-low risk; analyst/review queue.
0.50 <= probability < model_thresholdFLAGGEDSuspicious; step-up authentication or manual review.
probability >= model_thresholdBLOCKEDHigh risk; block and investigate.
Current active XGBoost threshold is 0.999. The lower alert save threshold defaults to 0.10.

11. Command Prompt Runbook

Use Command Prompt from the project folder:

cd /d "C:\Users\admin\Desktop\Real-Time Fraud Detection System"
venv\Scripts\activate.bat

Install or repair packages:

python -m pip install --upgrade pip setuptools wheel
pip install -r requirements.txt

Build/train/tune:

python -m src.processing.feature_engineering
python -m src.modeling.train_batch
python -m src.modeling.threshold_tuner

Start FastAPI:

uvicorn src.api.main:app --host 127.0.0.1 --port 8001 --reload

Start Streamlit in a second Command Prompt:

streamlit run app.py --server.port 8502

13. Known Limitations

  • The Kaggle dataset features V1 to V28 are PCA-anonymized, so SHAP explanations are mathematical drivers, not direct business reasons.
  • Kafka/Spark streaming exists in the project, but the easiest Windows demo path is the local demo generator.
  • The reviewer zip excludes raw data, model binaries, validation CSV, parquet, venv, and MLflow runs to keep it small and shareable.
  • Static model artifacts may show old training thresholds; the dashboard now labels or replaces threshold-specific artifacts with dynamic active-threshold views.
